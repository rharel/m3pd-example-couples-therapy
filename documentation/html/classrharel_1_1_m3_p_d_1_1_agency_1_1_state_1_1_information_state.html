Detailed Description

The information state is an object that is shared among all operation modules.

The information state is to be used as a container for any data that is read or written by more than one module.

The state organizes its data into "components". Each component is associated with a type and an identifier. Which components are available on a given state object is specified during construction, and can be queried through the ComponentIDs property.

Components identifiers prefixed with an _underscore should be considered reserved.
